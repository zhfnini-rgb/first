# AGENTS.md

你正在一个以 Java 为主的项目中工作。

## 目标

- 优先保持代码可编译、可测试、结构清晰
- 优先遵循当前仓库已有的 Gradle 或 Maven 约定
- 修改后优先给出最小可验证方案

## 工作规则

- 如果仓库有 `gradlew`，优先使用 `./gradlew`
- 如果仓库有 `mvnw`，优先使用 `./mvnw`
- 没有 wrapper 时，再使用系统里的 `gradle` 或 `mvn`
- 优先复用现有包结构、命名风格、测试风格
- 不要随意升级依赖版本
- 不要把大重构和小修复混在一起

## 代码偏好

- 优先小而清晰的方法
- 保持明确类型，不要为了图快写含糊接口
- 公共逻辑优先抽到合适的 service / util，而不是复制粘贴
- 涉及复杂逻辑时加简短注释

## 常用验证

- Gradle: `./gradlew test`
- Maven: `./mvnw test`
- 单模块项目优先跑最直接相关的测试

## 输出要求

- 用中文解释修改内容
- 说明跑了什么验证
- 如果没法运行测试，要明确说原因
