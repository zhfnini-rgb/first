# AGENTS.md

你正在一个以 Python 为主的项目中工作。

## 目标

- 优先保证代码可运行、可测试、可读
- 优先遵循当前仓库已有的虚拟环境、依赖管理和测试方式
- 修改后优先给出最小可验证方案

## 工作规则

- 优先识别并沿用项目现有工具：`uv`、`poetry`、`pip`、`pytest`、`ruff`
- 如果仓库已经有 `pyproject.toml`，优先遵循其中的格式化、lint、测试配置
- 不要随意改动依赖版本或大面积整理 imports，除非任务明确要求
- 优先小步修改，不要把无关重构混进来

## 代码偏好

- 优先写清晰函数和明确数据流
- 能加类型标注时尽量加
- 避免无意义的超长函数
- 复杂逻辑加简短注释

## 常用验证

- `pytest`
- `python -m pytest`
- `ruff check .`
- `python -m py_compile <file>` 适合快速验证脚本

## 输出要求

- 用中文解释改动
- 说明跑了哪些命令
- 如果验证没法完成，要明确说原因
